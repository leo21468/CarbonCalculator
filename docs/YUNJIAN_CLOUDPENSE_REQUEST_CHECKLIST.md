# 向云简业财（Cloudpense）索要接口与认证 — 对接清单

本文档供与云简商务/实施/客户经理沟通时使用，便于一次性明确 **认证**、**接口范围** 与 **实施条件**。背景：企业在云简做费控/业财记账时，**同步将发票或费用明细交给外部系统做碳足迹核算**（与本项目 `docs/ENTERPRISE_INTEGRATION.md` 中的集成目标一致）。

## 一、认证与接入资格（优先确认）

| 序号 | 需向云简确认/索要 | 说明 |
|------|-------------------|------|
| 1 | **开放平台 / OpenAPI 是否开通** | 是否在合同或模块中包含接口对接权限。 |
| 2 | **`client_id`、`client_secret`** | 用于调用 `common/unAuth/tokens/generate`（POST）换取 `access_token`；问清 **测试与生产是否各一套**。 |
| 3 | **实例所在云与 Base URL** | AWS / 华为云 / 阿里云对应 **测试、生产** 的 openapi 根地址（以对方书面确认为准，勿仅凭文档表自行猜测）。 |
| 4 | **调用约束** | 限流（全局/单接口）、是否 **IP 白名单**、HTTPS 要求等。 |
| 5 | **多法人 / 多账套** | API 中 **公司编码、账套、组织** 等字段名称、是否每笔请求必传、如何与贵司主数据对齐。 |

### TOKEN 使用要点（文档摘要）

- 接口：`POST {base}/common/unAuth/tokens/generate`  
- Body：`client_id`、`client_secret`（JSON）  
- 返回：`access_token`、`expires_in`（常见 7200 秒）、`token_type`、`scope` 等  
- 业务请求需按文档将 **`access_token` 置于请求 Params**；可配合请求头 **`locale`**（如 `zh_CN`）  
- 文档建议：**缓存 token 约 6000～6600 秒**，避免频繁申请；注意 **获取 TOKEN 接口自身限流**（文档示例为约 60 次/小时）

## 二、业务接口 — 按「碳核算集成」场景索要

请对方提供 **接口文档章节名称 + 是否已授权 + 示例报文（脱敏）**。

### A. 输入侧：发票 / 费用明细（供碳核算）

| 需求 | 建议向对方要的接口类型 |
|------|------------------------|
| 电子发票结构化数据 | 发票查询、下载或 **推送回调**；是否含 PDF/OFD/XML 或已解析字段。 |
| 报销 / 对公 / 采购等单据 | 单据及 **明细行**（品名、金额、数量、单位、**税收分类编码**若有）。 |
| 与凭证关联 | 按 **单据号、凭证号、会计期间** 查询或关联上游明细的能力说明。 |

### B. 输出侧：碳结果是否写回云简（可选但建议早定）

| 需求 | 建议向对方要的接口或能力 |
|------|---------------------------|
| 回写核算状态 / 数值 | 是否支持 **自定义字段**、备注、附件上传等写入接口。 |
| 事件驱动 | 是否支持 **Webhook / 回调**（贵司提供 URL，云简推单据状态变更）。 |
| 若无法写回 | 明确业财侧是否仅保留 **外链或手工字段**，碳数据以 **外部系统为准** 的架构是否可接受。 |

### C. 主数据（常见卡点）

- 供应商、客户、成本中心、项目、部门等 **查询或同步** 接口。  
- 与排放归属（组织/项目维度）一致的 **编码体系** 说明。
